# Endpoint Investigation: dc — UAC Bypass via Scheduled Task Followed by Discovery on Domain Controller

**Date:** 2026-06-28  
**Host:** `dc`  
**Timerange:** -24h  
**Severity:** High  
**Assignee:** David  
**Case / Ticket:** [[CASE-0001]]  
**Data Sources:** Wazuh (10,000+ events)

## Executive Summary

The domain controller `dc` (agent IP: 192.168.1.50) shows a compact but high-confidence attack sequence beginning at 00:46 UTC on 2026-06-29. Task Scheduler (`svchost.exe -k netsvcs -p -s Schedule`) launched `explorer.exe /NOUACCHECK` as `CONDEF\Administrator` — a known UAC bypass technique — triggering a Wazuh level-12 Sysmon alert. Sixteen seconds later, `whoami` was executed in a `cmd.exe` shell under the same account, confirming the attacker was verifying their elevated context post-bypass. The surrounding 24-hour window contains approximately 10,000+ events, the overwhelming majority of which are Splunk forwarder noise (splunkd.exe → cmd.exe → btool.exe chains) that should be tuned or suppressed. The two events at 00:46 UTC are the critical signal: a UAC bypass on the CONDEF domain controller by the domain Administrator account, followed by discovery activity, represents a high-risk foothold on the highest-value asset in the environment.

### Suspicious Activity

#### UAC Bypass via Scheduled Task — explorer.exe /NOUACCHECK

- **Description:** `explorer.exe` was launched with the `/NOUACCHECK` flag by the Windows Task Scheduler service (`svchost.exe -k netsvcs -p -s Schedule`) as `CONDEF\Administrator`. This is a recognized UAC bypass pattern where a scheduled task runs a trusted process with UAC elevation suppressed, granting an elevated shell without a consent prompt.
- **Event IDs:** EID 1 (Sysmon Process Create)
- **Count:** 1
- **Key Indicators:**
  - Timestamp: `2026-06-29T00:46:15.779Z`
  - User: `CONDEF\Administrator`
  - Image: `C:\Windows\explorer.exe`
  - Command: `"C:\Windows\Explorer.EXE" /NOUACCHECK`
  - Parent: `C:\Windows\system32\svchost.exe -k netsvcs -p -s Schedule`
  - Wazuh Rule: 61640 (Level 12)
- **ATT&CK:** [[T1548.002]] — Abuse Elevation Control Mechanism: Bypass User Account Control

#### Scheduled Task as Execution Vector

- **Description:** The parent of the UAC bypass is the Task Scheduler service (`svchost -k netsvcs -p -s Schedule`), indicating a scheduled task was created or pre-existing and triggered to execute this UAC bypass. No EID 4698 (task creation) was observed in the current window, suggesting the task may have been created earlier or outside the 24h timerange.
- **Event IDs:** EID 1 (Sysmon, parent chain), EID 4698 (not seen — investigate)
- **Count:** 1 observed trigger
- **Key Indicators:**
  - Parent process: `svchost.exe -k netsvcs -p -s Schedule` (Task Scheduler host process)
  - Execution context: `CONDEF\Administrator`
  - Timestamp: `2026-06-29T00:46:15.779Z`
- **ATT&CK:** [[T1053.005]] — Scheduled Task/Job: Scheduled Task

#### Post-Exploitation Discovery — whoami

- **Description:** `whoami` was executed in an interactive `cmd.exe` session as `CONDEF\Administrator` 16 seconds after the UAC bypass. This is a textbook post-exploitation identity confirmation step.
- **Event IDs:** EID 1 (Sysmon), EID 4688 (Security)
- **Count:** 1
- **Key Indicators:**
  - Timestamp: `2026-06-29T00:46:31.930Z`
  - User: `CONDEF\Administrator`
  - Command: `whoami`
  - Parent: `"C:\Windows\system32\cmd.exe"` (PID 6912)
  - Process PID: 7056
  - Wazuh Rule: none triggered directly; captured in archives
- **ATT&CK:** [[T1033]] — System Owner/User Discovery

#### cmd.exe Execution by Scheduled Task Chain

- **Description:** `cmd.exe` was present as the parent of `whoami`, indicating an interactive command shell was obtained, likely as a result of or concurrent with the explorer.exe UAC bypass. The scheduler → explorer → cmd.exe chain is a known technique to obtain an elevated interactive shell without a UAC prompt.
- **Event IDs:** EID 4688, EID 1
- **Count:** 1 (interactive shell confirmed)
- **Key Indicators:**
  - User: `CONDEF\Administrator`
  - Parent command line: `"C:\Windows\system32\cmd.exe"` (PID 6912)
  - Timestamp window: 00:46:14–00:46:31
